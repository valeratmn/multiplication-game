# Игра на умножение

Интерактивная игра для изучения таблицы умножения, написанная на **TypeScript**.

## Описание

Игра позволяет пользователю тренироваться в умножении на заданное число N (по умолчанию 5). Интерфейс включает:
- Визуальную схему кубиков как подсказку
- Анимацию зеленых блоков, вылетающих снизу к схеме
- Интерактивное поле ввода для ответов
- Кнопку проверки с различными состояниями

## Функциональность

- Генерация случайных примеров на умножение
- Визуальная подсказка в виде схемы кубиков
- Анимация блоков, летящих к схеме (500ms)
- Проверка правильности ответов
- Визуальная обратная связь (зеленый/красный цвет)
- Автоматический переход к следующему примеру

## Состояния кнопки

- **disabled** (#A9A9A9) - кнопка неактивна
- **active** (#2DABA6) - кнопка активна (когда введено число)
- **hover** (#AFEEEE) - реакция при наведении мыши
- **right** (#228B22) - правильный ответ
- **wrong** (#FA8072) - неправильный ответ

## Установка и запуск

### Требования
- Node.js (версия 16 или выше)
- npm или yarn

### Установка зависимостей

```bash
npm install
```

### Сборка проекта

```bash
# Однократная сборка
npm run build

# Сборка в режиме наблюдения (watch mode)
npm run watch
```

### Запуск

После сборки просто откройте файл `index.html` в браузере. Для разработки рекомендуется использовать локальный сервер (например, `live-server` или встроенный сервер VS Code).

### Как это работает

1. При загрузке появляется первый пример слева: `4 × 1 = [?]`
2. Справа показываются 4 зеленых кубика (подсказка)
3. Снизу вылетают 4 зеленых блока к кубикам (500ms анимация)
4. После анимации инпут разблокируется
5. Введите ответ и нажмите "Done" или Enter
6. При правильном ответе - инпут зеленеет, появляется новый пример **под текущим**
7. При неправильном - инпут краснеет на 1000ms, значение стирается
8. Примеры накапливаются друг под другом

## Деплой на GitHub Pages

Проект настроен для автоматического деплоя на GitHub Pages через GitHub Actions.

### Автоматический деплой

1. Убедитесь, что репозиторий находится на GitHub
2. Перейдите в **Settings** → **Pages** вашего репозитория
3. В разделе **Source** выберите **GitHub Actions**
4. При каждом push в ветку `main` или `master` проект автоматически соберется и задеплоится

### Ручной деплой

Если нужно задеплоить вручную:
1. Перейдите в раздел **Actions** вашего репозитория
2. Выберите workflow **Deploy to GitHub Pages**
3. Нажмите **Run workflow**

После успешного деплоя ваш сайт будет доступен по адресу:
`https://<ваш-username>.github.io/<название-репозитория>/`

## Архитектура

Код организован в модули по принципу разделения ответственности с полной типизацией:

- **types.ts** - все типы и интерфейсы проекта
- **config.ts** - конфигурация и настройки
- **state.ts** - управление состоянием игры
- **dom.ts** - работа с DOM элементами (класс с проверками)
- **example-generator.ts** - генерация и отображение примеров
- **cubes-scheme.ts** - создание и управление схемой кубиков
- **blocks-animation.ts** - анимация летящих блоков
- **button-state.ts** - управление состояниями кнопки
- **answer-handler.ts** - обработка и валидация ответов
- **ui.ts** - UI компоненты и сообщения
- **game-controller.ts** - главный контроллер, координирующий работу всех модулей
